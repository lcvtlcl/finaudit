# Пакетный анализ документов («калькулятор» сведения)

Юзер грузит пачку файлов → сервис распознаёт их, сводит совместимые в один анализ,
про несовместимые пишет, почему их нельзя объединить.

## Что считаем совместимым

«Свести воедино» = это выписки одного счёта/потока за разные периоды.
Ключ совместимости — **счёт** (для 1С — `РасчСчет`; для CSV/XLSX счёт часто не указан → ключ «не определён»).

- Файлы с **одним** счётом → объединяются в один датасет (с дедупликацией пересечений периодов).
- Файлы с **разными** счётами → это разные потоки: анализируются отдельно, и в отчёте пишем
  «ФайлA не сводится с ФайлB в общий анализ: разные счета (…против…)».
- Файлы без счёта (CSV/XLSX) → группа «счёт не определён», объединяются между собой с пометкой,
  что счёт не подтверждён (риск смешать два бизнеса — предупреждаем).
- Битый/непарсируемый файл → статус «не распознан», в свод не идёт.

## Как сводим (merge)

Для каждой совместимой группы: склеиваем транзакции всех файлов → **дедуп**
(дата+сумма+контрагент+назначение, уже есть в `ingest`) → сортируем по дате →
прогоняем весь конвейер (метрики, чек-лист, налоги, **прогноз вперёд**) → один `AuditResult` на группу.
Так из четырёх квартальных выписок собирается годовая картина.

## API

- `POST /api/batches` (multipart, несколько `files`) → парсит каждый, группирует по счёту,
  сводит и анализирует каждую группу. Ответ: список файлов со статусами + группы с результатами + заметки о несовместимости.
- `GET /api/batches` / `GET /api/batches/{id}` — список и детали (если храним; см. объём).

## Хранение — два варианта объёма

- **MVP (быстро):** без новых таблиц. Загрузка обрабатывается за один запрос, результат отдаётся сразу
  (по группам). История пачек — позже.
- **Полный:** таблицы `batches` + `batch_files` (user_id, filename, account_key, период, tx_count, status, note),
  свод складываем как upload на группу. Тогда пачки видны в «Истории».

## UI

Экран «Пакет»: drag пачки файлов → таблица (файл · тип · период · счёт · статус) →
блок несовместимостей → под каждой группой — знакомый дашборд-результат (переиспользуем `render`).

## Дисциплина скоупа

Только выписки (CSV/XLSX/1С). PDF и отчёты маркетплейсов — отдельные роадмап-парсеры.
Сведение **разнотипных** документов (счета×акты, первичка) — это отложенная сверка (p6/p7), НЕ сюда.
