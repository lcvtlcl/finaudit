# Парсеры банков + роадмап типов документов

## Главный вывод (из ресёрча)

Все крупные банки РФ — **Сбер, Тинькофф, Альфа, ПСБ, Точка, Модульбанк** — отдают выписку в формате
**1CClientBankExchange** (`.txt`, «выгрузка для 1С», `kl_to_1c.txt`). Это де-факто стандарт обмена «клиент-банк».
Формат стандартизирован, но у банков есть **незначительные отличия** (имена полей, шапка, кодировка, архив).

**Следствие:** не нужно 5 парсеров под каждый банк. Нужен **один робастный 1С-парсер** + покрытие вариаций.
Это главный и самый дешёвый путь — им закрываются все банки сразу.

## Стратегия парсеров (по приоритету)

### 1. 1С-обмен (`.txt`) — универсальный, приоритет №1
Один парсер = все банки. Харднинг под реальные вариации:
- имена полей: `Получатель`/`Получатель1`, ИНН-варианты — **частично сделано**;
- шапка: `РасчСчет` в корне vs внутри `СекцияРасчСчет`;
- кодировки Windows-1251 / UTF-8 / CP866 — **есть**;
- **Сбер** часто отдаёт файл в **ZIP-архиве** → добавить распаковку;
- прогнать на реальных выгрузках Сбер/Тинькофф/Альфа/ПСБ (нужны образцы — см. блокер).

### 2. CSV/XLSX — реестр маппинга колонок (а не парсер на банк)
Банки экспортируют Excel/CSV с **разным расположением колонок**. Решение — не отдельный парсер,
а **таблица алиасов**: банк → имена его колонок → канонические поля (дата, сумма, дебет/кредит, контрагент, ИНН, назначение).
Один named-column парсер + маппинг покрывает Сбер/Тинькофф/Альфа/ПСБ CSV/Excel.
Авто-детект разделителя (`;`/`,`/таб) — **сделано**.

### 3. PDF — текстовый layout под банк (позже)
`pdftotext`/pdfplumber + правила под каждый банк; сканы/фото — vision-LLM как fallback (без обучения своей модели).

## Блокер

Чтобы настраивать и тестировать шаблоны, нужны **реальные (обезличенные) образцы выгрузок** каждого банка
(1С-txt, CSV, XLSX, PDF). Их не собрать массово (приватность). План: попросить у знакомых предпринимателей
по 1 обезличенной выписке на банк + использовать публичные образцы (уже собраны в `testdata/samples/open_sources`).

## Роадмап типов документов (что берём дальше)

**Tier 1 — деньги (ядро):**
- Банковские выписки — **готово** (CSV/XLSX/1С), PDF в роадмапе.
- Отчёты маркетплейсов (WB/Ozon/Яндекс.Маркет выплаты) — доход e-commerce.

**Tier 2 — первичка → сверка:**
- УПД, счёт-фактура, акт выполненных работ, ТОРГ-12 → движок сверки «выписка × документы» (что оплачено/чего не хватает).

**Tier 3 — бухотчётность → валидация (сильный демо-аргумент):**
- ОСВ (оборотно-сальдовая ведомость), бухбаланс, ОФР (отчёт о фин. результатах) → сверка нашей подбивки с «1С-годовым» («сходится до рубля»).

**Tier 4 — налоги:**
- Декларации УСН/НДС (XML/PDF ФНС) → проверка соответствия, база под compliance-модуль.

**Tier 5 — касса/зарплата:**
- ПКО/РКО, Z-отчёты ККТ, зарплатные ведомости.

**Приоритет:** добить Tier 1 (маркетплейсы) → Tier 3 (ОСВ для сверки — козырь на защите) → Tier 2 (сверка первички).
Налоги и касса — дальше. Каждый tier открывает новый сегмент и усиливает compliance/аналитику.
