# FinAudit — питч и демо-нарратив (защита)

Регламент прикинуть под формат Kodik (обычно 3–5 мин выступление + демо + Q&A).
Структура: **проблема → решение → демо → стек → roadmap**. Ниже — тезисы и сценарий.

## 0. Хук (15 сек)

> «Большинство малых бизнесов в России работают без бухгалтера-аналитика. Они видят
> остаток на счёте, но не видят кассовый разрыв за месяц до него. FinAudit — это
> финансовый аудитор, который читает банковскую выписку и говорит человеческим языком,
> где проблема и что делать.»

## 1. Проблема (30–40 сек)

- У МСБ нет штатного финаналитика; бухгалтер (если есть) считает налоги, а не риски.
- Банковские приложения показывают *прошлое* (остаток, обороты), но не *будущее*
  (когда не хватит денег) и не *риски* (налоги, контрагенты, обналичка).
- Существующие сервисы либо требуют долгой настройки/интеграции с 1С, либо это
  «GPT-обёртки», которым нельзя доверять цифры.

## 2. Решение (40–50 сек)

FinAudit: загрузил выписку (CSV / Excel / выгрузка в 1С / текстовый PDF) — получил аудит.

Ключевой принцип и наш moat: **все цифры считает детерминированный код** (аудируемо,
воспроизводимо, «сходится с 1С до рубля»), **а ИИ только объясняет** результат простым
языком. Мы не «обёртка над GPT».

Что внутри (всё детерминированно):
- кассовый разрыв и **прогноз баланса вперёд** по регулярным платежам;
- налоговый модуль (резерв УСН, порог НДС-2026);
- **правовые риски** со ссылками на реальные статьи (НДС/УСН/115-ФЗ/ст. 54.1);
- **проверка контрагентов по ИНН** (ЕГРЮЛ/ЕГРИП: ликвидация, банкротство, массовый адрес);
- рейтинг финансового здоровья, песочница «что если», сводка для руководства;
- приватность: self-host, свой ключ ИИ — данные не покидают периметр.

## 3. Демо (90–120 сек) — сценарий

Готовим заранее одну «сочную» выписку (есть кассовый разрыв + рисковый контрагент).

1. **Загрузка.** Перетаскиваем выписку → «Анализировать». Пока считает — видна анимация
   шагов (разбор → метрики → риски → выводы). *«Обратите внимание: считает наш код, не ИИ.»*
2. **Сводка для руководства.** Первым делом — светофор 🔴/🟡/🟢 + вердикт одной фразой
   + «главное на контроль». *«Директору не нужны таблицы — нужен вывод.»*
3. **Кассовый разрыв.** Показываем дату разрыва и сумму нехватки на графике прогноза.
   *«Приложение банка вам этого не скажет.»*
4. **Правовые риски + контрагент.** Флаг по ИНН (например, «контрагент в стадии
   ликвидации») со ссылкой на статью. *«Это должная осмотрительность, автоматически.»*
5. **Песочница «что если».** Жмём сценарий (перенести платёж) → код пересчитывает →
   «было/стало». *«Решение проверяемо, а не на глаз.»*
6. **Экспорт в Excel** и упоминание self-host. *«Данные остаются у клиента.»*

Запасной вариант на случай падения стенда — заранее записанное видео/скриншоты (d6c).

## 4. Стек (20–30 сек)

- Backend: Go 1.25, модульный монолит, chi + pgx, PostgreSQL.
- Детерминированный конвейер: ingest → categorize → metrics → forecast → checks →
  compliance → rating → llm (только объяснение).
- ИИ: DeepSeek (OpenAI-совместимый), ключ пользователя, AES-GCM для секретов.
- Инфра: Docker Compose, Caddy (авто-TLS), деплой с версионированием (`/version`).
- Внешние данные: DaData (ЕГРЮЛ/ЕГРИП) для контрагентов.

## 5. Roadmap (20 сек)

- Харднинг парсеров на реальных банковских выгрузках (Сбер/Тинькофф/Альфа/ПСБ).
- Сверка с первичкой (ОСВ/оборотка) — «сходится с 1С-годовым».
- Точность метрик: внутренние переводы/займы, сторно.
- MCP-сервер: запрос к своим финансам прямо из ИИ-ассистента.

## Ответы на вероятные вопросы (Q&A)

- **«Чем вы лучше банка/1С?»** — банк показывает прошлое, мы прогнозируем будущее и
  проверяем риски; 1С требует настройки, мы работаем от одной выписки за минуту.
- **«Почему вам можно верить цифрам, если это ИИ?»** — цифры считает код, ИИ только
  пишет текст; логику можно открыть и проверить, результат воспроизводим.
- **«Как с приватностью?»** — self-host + свой ключ ИИ; сырые выписки в модель не уходят.
- **«Монетизация?»** — подписка SaaS + self-hosted-лицензия для чувствительных клиентов.
