# Compliance-движок: проверка на признаки правового риска

Принцип: **решает код по правилам, привязанным к статьям. ИИ из принятия решения исключён**
(иначе он «придумает» нарушение). ИИ — максимум для финального пересказа, с фактами на входе, или вообще не участвует.

## Контракт правила

Каждая проверка — детерминированное правило:

```
ComplianceFlag {
  Code           // машинный код
  Title          // человеческий заголовок
  Severity       // ok | attention | risk
  Detail         // что именно посчитано (с числами)
  Statute        // ссылка на норму: "НК РФ ст. 346.13"
  Recommendation // что сделать
}
```

Правило срабатывает по условию на данных (транзакции + метрики + данные по ИНН). Норма (`Statute`) — **только цитата/ссылка для аудируемости**, не «мозг» решения.

## Правила из выписки (без внешних данных) — делаем первыми

- **Порог НДС для УСН** — годовой доход (аннуализированный) ≥ 20 млн ₽ → обязанность платить НДС. `НК РФ` (с 2026).
- **Лимит УСН** — доход ≥ 490,5 млн ₽ → потеря права на УСН. `НК РФ ст. 346.13`.
- **Обналичивание (115-ФЗ)** — доля снятия наличных выше порога → признак по антиотмывочному. `ФЗ-115`, методички ЦБ `18-МР/19-МР`.
- **Смешение личного/бизнеса, вывод на физлиц** — переводы на карту владельца/физлицам без основания → риск переквалификации, НДФЛ+взносы. `НК РФ ст. 54.1`.
- **Налоговая нагрузка ниже среднеотраслевой** — налоги/оборот vs бенчмарк ФНС. `Приказ ФНС ММ-3-06/333@` (12 критериев). *(нужен бенчмарк по отрасли — позже)*

## Правила по ИНН (внешние реестры) — `inn1`, позже

- Статус контрагента в ЕГРЮЛ/ЕГРИП (ликвидация/банкротство), массовый адрес, дисквалификация → должная осмотрительность `НК РФ ст. 54.1`. Данные — ФНС/DaData, не ИИ.

## Роль `lex1` (парсинг кодексов)

Библиотека цитат: по коду правила достаём точный текст статьи для показа. Не для рассуждений.

## Роль ИИ — минимальная

Либо **ноль** (тексты флагов пишет код), либо **только пересказ** готовых структурных findings + цитаты, с жёсткой инструкцией «не добавлять факты». ИИ не решает, что нарушено.

## Честная рамка

Не «проверка законности» / «гарантия», а **«признаки правового риска по проверяемым нормам»**, с указанием статьи и дисклеймером «не юридическая консультация». Каждый флаг аудируем (в отличие от «ИИ сказал»).

## Задачи

- `comp1` [A] — движок `internal/compliance` + правила из выписки (НДС/УСН/обналичка/смешение) с цитатами статей, в `AuditResult` + на дашборд.
- `comp2` [D] — правила по ИНН (ЕГРЮЛ/DaData) → флаги контрагентов (зависит от `inn1`).
- `lex1` [D] — база цитат статей (для показа ссылок).
- `mr2` [A] — сводный отчёт для руководства использует эти флаги (без ИИ в решении).
