# FinAudit — self-hosted развёртывание

FinAudit можно развернуть **на своей инфраструктуре**: данные выписок, результаты анализа
и ключ ИИ остаются в вашем периметре и не покидают его. Это вариант для тех, кому важна
максимальная приватность (бухгалтерия, чувствительные финансы).

## Что нужно

- Сервер с Docker и Docker Compose (Linux, 2 vCPU / 2 ГБ RAM достаточно для одной компании).
- Своя база данных PostgreSQL (можно поднять в том же compose — включена в комплект).
- Ключ ИИ-провайдера (DeepSeek или совместимый) — **ваш собственный**, обращения к модели
  идут в ваш аккаунт у провайдера, мы посредником не являемся.
- (Опционально) токен DaData для проверки контрагентов по ИНН.

## Развёртывание за 4 шага

### 1. Получить код

```
git clone <репозиторий> finaudit && cd finaudit
```

### 2. Настроить окружение

Скопируйте пример и заполните значения:

```
cp .env.selfhosted.example .env
```

Обязательные переменные:

| Переменная         | Назначение                                                                 |
|--------------------|----------------------------------------------------------------------------|
| `POSTGRES_DSN`     | Строка подключения к вашей БД (`postgres://user:pass@host:5432/finaudit`)   |
| `POSTGRES_PASSWORD`| Пароль БД (если поднимаете Postgres из комплекта) — задайте сильный         |
| `APP_ENC_KEY`      | Ключ шифрования секретов (32 байта base64): `openssl rand -base64 32`       |

Опциональные:

| Переменная         | Назначение                                                                 |
|--------------------|----------------------------------------------------------------------------|
| `DEEPSEEK_API_KEY` | Глобальный ключ ИИ (можно не задавать — пользователи вводят свой в кабинете)|
| `DADATA_API_KEY`   | Токен DaData для проверки контрагентов по ИНН (иначе этот блок не активен)  |

`.env` **не коммитьте** — он содержит секреты.

### 3. Применить миграции и запустить

```
docker compose -f deploy/docker-compose.selfhosted.yml up -d --build
# накатить схему БД (goose up) — см. deploy/README
```

### 4. Проверить

```
curl -s http://localhost:8080/health     # {"status":"ok"}
curl -s http://localhost:8080/version    # покажет версию сборки
```

Откройте `http://<ваш-хост>:8080` (или свой домен через reverse-proxy/Caddy),
зарегистрируйте первого пользователя — готово.

## Приватность и данные

- Все финансовые расчёты выполняет код на вашей стороне; в модель ИИ уходят только
  агрегированные метрики, а не сырые выписки.
- Ключи провайдеров и пароли хранятся зашифрованными (AES-256-GCM / bcrypt).
- Сырые файлы и БД — в вашем хранилище; мы к ним доступа не имеем.

## Обновления и бэкапы

- Обновление: `git pull`, затем пересборка `docker compose ... up -d --build`.
- Бэкап: регулярно снимайте дамп PostgreSQL (`pg_dump`) и том с загруженными файлами.
- Проверить, какая версия развёрнута: `curl /version`.
